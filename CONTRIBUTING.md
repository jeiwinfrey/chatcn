# Contributing to chatcn

Thanks for helping improve chatcn.

## Before you open a PR

Run the checks locally:

```bash
npm run typecheck
npm test
npm run build
```

## What to include

- A clear description of the problem or feature
- A minimal reproduction, if applicable
- Tests for behavior changes
- Notes about framework or provider impact

## Style

- Keep changes focused and small
- Prefer beginner-friendly wording in user-facing text
- Preserve the current CLI and template conventions unless the change is intentional

## Reporting issues

Open an issue on GitHub:

https://github.com/jeiwinfrey/chatcn/issues
